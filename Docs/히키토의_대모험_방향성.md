# 히키토의 대모험 — 게임 디자인 방향성

> **장르:** Dynamic Adventure MMORPG
> **제작 방식:** AI-Only 제작
> **최종 수정:** 2026-03-31

---

## 1. 비전

**"AI가 끊임없이 세계를 확장하고, 플레이어가 미지의 세계를 함께 탐험하며, 베풂과 발견의 쾌감을 나누는 살아있는 모험 세계"**

---

## 2. 설계 기둥 (4 Pillars)

| 기둥 | 핵심 감정 | 한 줄 정의 |
|------|-----------|------------|
| 성장하는 세계 | 경이, 몰입 | 플레이어 행동에 반응해 변이하는 생태계 |
| 모험심 | 긴장, 발견 | "다음에 뭐가 나올지 모른다"는 미지(未知)의 쾌감 |
| 성장/도파민 | 성취, 욕구 | 짧은 루프와 긴 루프가 겹치며 터지는 보상 |
| 협동(시혜) | 존재감, 우월감 | "내가 베풀어서 모두가 나아졌다"는 심리적 만족 |

---

## 3. 핵심 전제: AI 제약의 역전

AI-Only 제작이라는 제약은 곧 설계의 출발점이다.

**피해야 할 것 (AI가 약한 곳)**
- 고정된 정교한 에셋 하나의 퀄리티 경쟁 (AAA 그래픽, 수작업 컷씬)
- 수공예적 디테일에 의존하는 연출

**밀어야 할 것 (AI가 강한 곳)**
- 대량 생성, 변이, 조합의 폭발적 다양성
- 매번 다른 경험 — 절차 생성 던전, 지형, 몬스터 변이
- 텍스트/내러티브 깊이 — 모든 NPC가 고유한 성격과 기억을 가짐
- 데이터 기반 시스템(로그, 기록, 감사 시스템) — 에셋 부담 최소

---

## 4. Pillar 1 — 성장하는 세계

### 4.1 생태계 시뮬레이션 레이어

서버 단위 백그라운드 시뮬레이션이 세계를 구동한다.

- 플레이어가 특정 지역 몬스터를 과도 사냥 → 먹이사슬 변화 → 상위 포식자 이주
- 자원 풍부 지역 → 새로운 NPC 정착지 자연 발생
- AI가 지형 변이, 던전 레이아웃, NPC 대사/퀘스트를 절차적으로 생성

### 4.2 시간축 레이어링

같은 지역이라도 "계절"·"시대"에 따라 지형, 몬스터, 자원이 달라진다.
→ AI 텍스처 변이 + 프로시저럴 배치로 구현 가능

### 4.3 플레이어 집단 행동 → 세계 변화 피드백 루프

서버 전체 플레이어의 주간 자원 소비를 집계하여 다음 주기에 세계 상태를 갱신한다.
→ "우리가 이 세계를 바꾸고 있다"는 감각

### 4.4 비가역적 사건

돌이킬 수 없는 월드 이벤트(도시 함락, 새 대륙 부상)를 통해 "그때 거기 있었다"는 서버 히스토리를 생성하고, 커뮤니티 서사를 자연 발생시킨다.

---

## 5. Pillar 2 — 모험심

### 5.1 핵심 컨셉: "지도가 없는 세계"

- 플레이어가 직접 탐험한 곳만 지도에 기록
- 미탐험 영역은 매 접속 시 일부 재생성 (시드 기반 + 규칙 기반)
- "논리적이지만 예측 불가능한" 세계

### 5.2 발견 보상 시스템

- 새 지역, 생물, 아이템의 **최초 발견자**에게 칭호 + 명명권 부여
- 발견이 서버 전체 도감에 영구 등록
- AI가 끊임없이 새로운 변이체를 생성하므로 발견 대상이 고갈되지 않음

### 5.3 위험-보상 그래디언트

안전지대에서 멀어질수록:
- 보상이 기하급수적으로 증가
- 위험도 비례 증가
- **환경 규칙 자체가 변화** (중력, 시야, 자원 회복 속도 등)

### 5.4 귀환의 긴장감

- 먼 곳에서 얻은 보상은 안전하게 귀환해야 확정
- 사망 시 일부 유실
- 로그라이크적 긴장감을 오픈월드에 녹임

---

## 6. Pillar 3 — 성장 / 도파민

### 6.1 3단계 루프 구조

| 루프 | 시간 | 내용 | 도파민 포인트 |
|------|------|------|---------------|
| 마이크로 | 30초~5분 | 전투 한 판, 자원 채집 | 크리티컬 발견 (저확률 고보상) |
| 미들 | 30분~2시간 | 던전 클리어, 퀘스트 완료 | 예상 못한 보너스, 숨겨진 방 |
| 매크로 | 주~월 | 빌드 완성, 거점 확장, 서버 이벤트 | 시각적 성장, 서버 랭킹 |

### 6.2 AI 동적 드롭 조절

- 플레이어의 현재 진행도 + 최근 보상 이력에 따라 "적절한 타이밍에 터지는" 경험을 AI가 설계
- 같은 보상이 두 번 나오지 않는 구조: 무기/장비에 AI 미세 변이(외형 패턴, 이름, 부가 효과 조합)

### 6.3 유니크 아이템 철학

모든 드롭은 "세상에 하나뿐인 아이템"이다.
→ 수집 욕구 자극 + 세밀한 에셋 제작 부담 없이 달성 (모듈러 조합 생성)

---

## 7. Pillar 4 — 협동 (시혜자 심리)

### 7.1 시혜자(施惠者) 심리의 3층 구조

이 게임의 협동 설계는 "같이 하면 효율적이다"가 아니라, **"내가 베풂으로써 느끼는 우월감과 존재감"**을 핵심 동기로 삼는다. 리니지 혈맹주, SLG 동맹 맹주가 대과금하는 심리의 재현.

| 층위 | 심리 | 게임 내 경험 |
|------|------|-------------|
| 1층: 자기효능감 | "내 힘으로 해결해줬다" | 길드원이 막힌 던전을 내가 뚫어줌 |
| 2층: 사회적 인정 | "형님 감사합니다" | 반복되는 감사 → 게임 내 권위 형성 |
| 3층: 불가결성 | "이 사람 없으면 안 된다" | 떠나면 길드가 흔들릴 포지션 → 이탈 락인 |

### 7.2 "베풂이 보이는" 시스템

**거점 후원 시스템**
- 길드 거점 시설(대장간, 회복소, 방어벽)을 특정 플레이어가 자원 투자로 업그레이드
- 시설에 후원자 이름 각인 → 이용 시마다 "○○님이 건설한 대장간" 표시
- 길드원 전원이 매일 그 이름을 봄

**장비 선물 / 제작 대행**
- 고레벨이 하위 플레이어에게 장비 제작·강화 가능
- 받는 쪽 장비에 "○○님의 축복" 각인 → 관계의 가시화

**원정 지원**
- 파티 탐험 시 후방에서 자원 보급 역할
- "지원 덕분에 클리어" 결과 로그가 길드 전체에 공유

### 7.3 "감사가 시스템화"되는 구조

**공헌 보드**
- 길드 거점에 이번 주 최대 공헌자의 동상/초상화 표시
- 캐릭터 외형 기반 동상을 AI가 절차 생성

**감사 리액션 시스템**
- 도움받은 플레이어가 원터치로 감사 발송 → "존경" 수치로 누적
- 존경 수치에 따른 해금: 특별 칭호, 캐릭터 오라 이펙트, 전용 이모트
- **돈으로 살 수 없고, 오직 남을 도와야만 획득 가능**

**길드 히스토리 로그**
- "3월 15일, ○○님의 지원으로 용의 둥지 최초 클리어" 영구 기록
- 길드의 역사 = 시혜자들의 기록

### 7.4 "없으면 안 되는 사람" 만들기

**희귀 전문성 시스템**
- 특정 스킬 트리/제작 레시피를 서버 내 소수만 보유
- "이 서버에서 전설급 무기 제작 가능한 사람은 세 명뿐" → 의뢰 쇄도 → 자연발생적 시혜자 포지션

**멘토 시스템**
- 고레벨이 저레벨을 멘토링 → 양쪽 보상
- 제자 성장 시 멘토의 명성 상승 → "내가 키운 사람"이라는 애착 → 사회적 관계망 형성

**위기 대응 역할**
- 월드 이벤트(재앙, 침공) 발생 시 고과금/고레벨의 결정적 역할 구조
- "○○님이 나서서 해결" 서사 → 서버의 영웅 → 돈으로 환산 불가능한 심리적 보상

## 8. 과금 모델: "베풂의 확장" 패러다임

### 8.1 핵심 철학

> **과금 = 나 자신의 강해짐이 아니라, 내가 남에게 줄 수 있는 것의 확대**

리니지 혈맹주의 현질 심리: "내가 강해지려고"가 아니라 "혈맹을 강하게 만들어주려고"

### 8.2 과금 아이템 방향

| 과금 항목 | 효과 | 시혜 심리 |
|-----------|------|-----------|
| 후원 확장 | 길드 시설 다수 후원 가능 | 더 많이 베풀 수 있음 |
| 선물 슬롯 확장 | 일일 장비/자원 선물 횟수 증가 | 더 자주 줄 수 있음 |
| 거점 건축 아이템 | 길드 거점의 시각적 품질 향상 | 길드원 전체 혜택 → 시혜 만족 극대화 |
| 이벤트 개최권 | 길드 내 미니 이벤트 직접 개최 + 보상 설정 | "내가 판을 깔아주는" 경험 |

### 8.3 설계 원칙

- 직접적 전투력 판매(P2W)는 피한다 → 반감 유발
- 과금은 "시혜 능력의 범위 확장"으로 포지셔닝
- 과금 유저가 강해지는 게 아니라, 과금 유저 **주변이** 강해지는 구조

## 9. AI-Only 제작 전략

### 9.1 비주얼 스타일

**스타일라이즈드 (셀셰이딩 / 복셀 / 일러스트레이티브)**
- 포토리얼은 AI 생성 에셋의 품질 편차가 눈에 띔
- 강한 아트 스타일이면 AI 생성물의 일관성 유지가 쉬움
- 절차 생성 지형의 반복감도 스타일로 흡수

### 9.2 모듈러 에셋 시스템

- AI로 "부품"을 대량 생성, 조합 규칙으로 최종 오브젝트 생성
- 검: 칼날 + 자루 + 장식 각각 생성 후 조합
- 변이의 폭 극대화, 개별 에셋 퀄리티 부담 최소화

### 9.3 텍스트 / 내러티브 (AI 최대 강점)

- NPC 대화, 퀘스트 텍스트, 아이템 설명, 세계 역사를 LLM 생성
- 모든 NPC가 고유 성격 + 플레이어와의 대화 맥락 기억
- 다른 게임이 따라올 수 없는 내러티브 깊이

### 9.4 사운드

- 환경음, 효과음을 AI 생성
- 지역/상황에 따라 동적 믹싱

### 9.5 시혜 시스템과의 시너지

- 감사 리액션, 공헌 보드, 히스토리 로그 → 데이터 기반 시스템 → 에셋 부담 거의 없음
- 공헌자 동상 → 캐릭터 메시 + 포즈 조합 절차 생성
- 길드 거점 시설 → 모듈러 에셋 조합으로 충분
- "감사 표현"과 "기록" → 텍스트 기반 = AI 강점 영역
